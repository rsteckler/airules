## Documentation Policy: Do Not Modify Unless Explicitly Requested

You must **not create, update, or modify any documentation or planning artifacts unless the user explicitly asks**.

### Scope
This includes:
- README files
- Markdown documentation
- architecture or design docs
- ADRs, plans, or roadmaps
- comments intended as documentation
- user or developer guides

### Default Behavior
- Focus only on code or configuration changes.
- Do not suggest documentation updates.
- Do not add or modify comments purely for documentation purposes.

### When Asked
If the user requests documentation changes:
- Update only the specified files or sections.
- Keep content concise and accurate.
- Reflect actual behavior, not intended or future behavior.

### Principle
Documentation changes are explicit product decisions and should not be made automatically.
